# Crewbe Interview Android 프로젝트 규칙

## 1. 프로젝트 구조

```
crewbe-interview-android/
├── src/
│   ├── types/         # TypeScript 타입 정의
│   ├── components/    # 재사용 가능한 컴포넌트
│   ├── screens/       # 화면 컴포넌트
│   ├── navigation/    # 네비게이션 관련 로직
│   └── utils/         # 유틸리티 함수
├── App.tsx            # 앱의 진입점
├── app.json          # Expo 설정
└── package.json      # 프로젝트 의존성 관리
```

## 2. 네비게이션 규칙

### 2.1 스택 구조
- RootStack: 앱의 최상위 네비게이션 스택
  - UserNameInput: 사용자 이름 입력 화면
  - MainTabs: 메인 탭 네비게이션

### 2.2 탭 구조
- Home: 홈 화면
- Profile: 프로필 화면
- Schedule: 일정 화면
- MockInterview: 모의 면접 화면

## 3. 코딩 컨벤션

### 3.1 TypeScript
- 모든 컴포넌트는 TypeScript로 작성
- 타입 정의는 `src/types` 디렉토리에 위치
- 네비게이션 타입은 `navigation.ts`에서 관리

### 3.2 컴포넌트
- 컴포넌트는 함수형 컴포넌트 사용
- Props는 명시적인 타입 정의 필수
- 재사용 가능한 컴포넌트는 `src/components`에 위치

### 3.3 네이밍 컨벤션
- 컴포넌트: PascalCase (예: `UserNameInput`)
- 함수/변수: camelCase (예: `handleSubmit`)
- 타입/인터페이스: PascalCase (예: `RootStackParamList`)
- 상수: UPPER_SNAKE_CASE (예: `MAX_RETRY_COUNT`)
